# 讯飞工作流平台 UI 配置契约

本文件只记录用户截图已经确认的页面字段。WF-01～WF-12 必须遵守本契约，不得用抽象概念替代页面配置。

## 1. 开始

在“输入”中点击“+ 添加”。每行填写变量名、变量类型、描述、是否必要。系统自带 `AGENT_USER_INPUT:String`；业务工作流必须另建 `uid:String`，不得把用户自然语言当 uid。

## 2. 大模型

- 模型：工作流统一选 `Spark4.0 Ultra`，除非文档明确说明其他已验证模型。
- 输入：参数名必须与提示词中的 `{{变量名}}` 完全一致；参数值选择“引用”并选上游输出。
- 系统提示词：角色、规则、安全边界、JSON 结构。
- 用户提示词：本轮输入和引用变量。
- 输出：格式 `text`，变量 `output:String`，填写描述。
- 对话历史：状态机工作流关闭，避免混入旧轮次；确需历史时必须解释原因。

## 3. 变量提取器

- 模型：`Spark4.0 Ultra`。
- 输入：`input｜引用｜上游大模型/output`。
- 输出：逐行添加变量名、类型和必填描述。它是模型提取，不是固定 JSON parser。
- 下游要进入代码节点时，完整对象优先选择 `Object`、数组选择 `Array`；只有当前页面确实没有 Object/Array 可选项时才输出 String。
- 当前代码运行环境按用户实测不得使用 `import`，因此不要写 `json.loads/json.dumps`。若只能得到 String，代码节点只做非空、枚举和首尾括号检查；需要字段级校验时，回到变量提取器把字段逐项提取出来。

## 4. 代码

- 仅支持 Python。
- 输入参数名必须与 `main(...)` 形参一致。
- 函数必须返回 dict。
- 输出区逐行声明返回 dict 的同名键及类型；不得只在代码中 `return` 而不声明输出。
- 不写任何 `import`；所有分支都必须返回同一组键，Integer 不得返回 `None`，缺省时用 `0`。

```python
def main(input):
    return {"output": input}
```

## 5. 分支器

用于 Boolean、String、Integer 等确定性变量判断。每条分支配置：引用变量、选择条件、比较类型、比较值；与字面量比较时选常量/固定值，不选引用。所有分支器必须有默认或失败出口。

## 6. 决策

页面包含模型、`Query`、意图名称、意图描述、默认意图、高级配置，输出 `class_name:String`。它只用于从自然语言做模型意图分类，不用于 `isSuccess == true` 等确定性判断。

## 7. 数据库：自定义 SQL

用于读取。选择数据库，输入区声明 SQL 模板参数，SQL 使用 `{{参数名}}`。固定输出：

```text
isSuccess Boolean
message String
outputList Array<Object>
```

`isSuccess=true && outputList=[]` 表示查询成功但无记录，不等于 SQL 失败。

## 8. 数据库：表单处理数据

### 更新数据

选择 `数据库 / 数据表`，处理模式“更新数据”。“设置数据范围”逐行填写表字段、条件、比较类型、比较值；“设置更新数据”逐行填写参数名、类型、值。

### 新增数据

选择 `数据库 / 数据表`，处理模式“新增数据”。在“设置新增数据”逐行添加参数名、类型、值。系统字段 `id/uid/create_time` 默认不重复填写；页面强制要求 uid 时才引用开始节点 uid。

两种模式固定输出 `isSuccess/message/outputList`。

## 9. 知识库

- 输入：`query:String`，参数值引用上游。
- 添加知识库后设置参数：Top K=`3`，Score 阈值=`0.20`。
- 调用逻辑：强制调用。
- 输出：`results:Array<Object>`。

## 10. 知识库 Pro

页面包含模型、Agentic RAG/Long RAG、`query`、知识库、回答规则；固定输出 `output:String` 和 `result:Array<Object>`。只有需要知识检索后直接总结回答时使用。

## 11. 问答节点

- 模型：`Spark4.0 Ultra`。
- 输入：参数值引用上游。
- 提问内容：可使用 `{{变量名}}`。
- 回答模式：直接回复。
- 输出：`query:String`（提问内容）、`content:String`（用户回答）。

## 12. 变量存储器

“设置变量值”通过输入行引用上游；“获取变量值”在输出区选择已经保存的参数。跨会话业务状态仍应写数据库，不把变量存储器当持久化数据库。

## 13. 文本处理节点

处理方式为字符串拼接或字符串分割；输入逐行引用；规则文本使用 `{{变量名}}`；输出 `output:String`。

## 14. 长期记忆

- 写入：`role` 选择角色，`content` 引用内容；输出 `isSuccess:Boolean,message:String`。
- 检索：`input` 引用查询文本，设置检索数量；输出 `memory:Array<Object>`。
- 本项目已有数据库表时优先数据库，避免同一业务事实双写。

## 15. 消息

输入逐行引用上游；“回答内容”使用 `{{变量名}}`；流式输出默认关闭。消息用于中间产物输出，所有终态消息必须连到共享结束节点。

## 16. 结束

- 回答模式：返回设定格式配置的回答。
- 调试阶段允许 `output｜输入｜workflow_finished`，但这只代表画布闭合，不是业务结果。
- 需要发布为 API 或被大工作流调用时，必须把终态业务结果先整理成上游 `result_json:String`，再配置 `output｜引用｜结果节点/result_json`；不能引用“消息”，因为消息节点没有可供结束节点选择的业务输出。
- 思考内容和回答内容可以引用 `{{变量名}}`、`{{变量名.子变量}}`、`{{变量名[索引]}}`。

## 17. 迭代

已确认输入为 Array、运行模式可选串行、错误时终止、输出可声明 Array；但当前项和内部节点映射界面未确认。因此 WF-01～WF-12 不使用迭代，改为数据库保存索引并跨调用推进。

## 18. 异常处理

初次搭建统一关闭。显式使用 `isSuccess`、Python 校验结果和分支器进入错误消息；不得依赖未配置的隐式异常路线。
