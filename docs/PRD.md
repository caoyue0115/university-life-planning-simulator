## 1. 产品概述

### 1.1 产品名称

大学人生规划模拟器。

### 1.2 产品定位

一个完全运行在讯飞星辰 Agent 平台内的游戏化大学规划智能体。产品先让用户以低成本体验不同选择可能带来的四年轨迹，再将模拟中的认识转化为真实、可执行、可持续修正的大学规划。

产品不是未来预测工具，而是面向大学阶段的情景推演、决策辅助、行动管理和成长档案系统。

### 1.3 核心价值

- 帮助用户理解保研、考研、就业、考公、留学五条路径的真实要求。
- 通过虚拟大学和平行人生降低试错成本，呈现选择的收益、风险与机会成本。
- 将长期目标拆解为学期、月度与每周行动。
- 根据成绩、活动、项目和实习等实际行为动态修正建议。
- 自动沉淀经历、技能证据和简历素材，减少毕业前集中整理的成本。
- 通过技能树、徽章、复盘和微习惯提升持续行动意愿。

### 1.4 产品原则

1. 用户拥有最终决策权，Agent 只提供地图、依据和选项。
2. 推荐必须可解释，同时提供备选路径与局限说明。
3. 游戏化用于降低认知门槛，不用于制造焦虑、排名或惩罚。
4. 画像区分用户陈述、Agent 推断和行为验证，允许查看、更正和删除。
5. 政策与时效性信息必须标注来源和更新时间，并提醒用户通过官方渠道复核。
6. MVP 必须形成完整闭环，而非只实现趣味测试。

### 1.5 统一免责声明

> 每个人的大学都是独一无二的。模拟器给的是“地图”，但“走路”的人是你自己。勇敢去闯，错了也没关系——毕竟，大学本身就是试错成本最低的地方呀！

## 2. 用户与场景

### 2.1 目标用户

- 大一：尚未确定方向，需要探索专业、能力与发展路径。
- 大二：需要收敛方向、建立项目和技能积累。
- 大三：需要聚焦升学、求职或考试目标，并建立备选方案。

大四用户可继续使用成果整理、申请、求职和复盘能力，但不是 MVP 的主要服务对象。

### 2.2 核心用户问题

- 不清楚不同发展路径分别需要哪些能力和履历。
- 选择过多，难以理解机会成本。
- 有目标但无法拆解成当前行动。
- 参与了活动和项目，却没有形成可复用的履历材料。
- 规划与真实行为脱节，方向变化后原计划无法更新。
- 每次咨询都要重复介绍背景，缺乏连续陪伴体验。

### 2.3 核心使用场景

1. 首次进入，建立个人画像并体验一轮虚拟大学。
2. 完成“大学生存大冒险”，获得路径与核心竞争力画像。
3. 创建 2～3 个平行版本，对比不同方向的剩余大学轨迹。
4. 存档一个版本为主规划，生成当前学期任务。
5. 持续记录成绩、活动、项目、比赛和实习，动态修正路线。
6. 完成事项后生成简历条目并保存证据位置。
7. 面对重大选择时进行机会成本分析或七天试错。
8. 会话结束时自动复盘，下次从保存状态继续。

## 3. 产品核心闭环

```text
建立画像
→ 用户确认初始属性
→ 虚拟大学快速试玩
→ 发现选择后果
→ 完成大学生存大冒险测试
→ 获得路径推荐
→ 创建 2～3 个平行版本
→ 对比轨迹、履历、技能、成本与风险
→ 存档主规划
→ 制定学期和每周任务
→ 执行、打卡、记录成果
→ 自动生成履历条目
→ 阶段复盘并动态修正
→ 进入下一轮规划
```

MVP 的关键成功体验是：用户一次完成建档、试玩、测试、平行人生对比并保存主规划；后续会话能够读取历史状态并继续执行与复盘。

## 4. 产品架构

### 4.1 平台边界

全部能力使用讯飞星辰 Agent 平台提供的主 Agent、工作流、数据库、知识库和可用插件实现，不建设外部小程序、独立网站或独立产品前端。

### 4.2 逻辑架构

```text
主 Agent（意图识别、对话编排、统一人设）
├─ WF-01 用户建档与画像更新
├─ WF-02 虚拟大学试玩
├─ WF-03 大学生存大冒险测试
├─ WF-04 路径推荐
├─ WF-05 平行人生模拟
├─ WF-06 主规划生成
├─ WF-07 学期任务管理
├─ WF-08 成长复盘
├─ WF-09 履历条目生成
├─ WF-10 决策分析与七天试错
├─ WF-11 微习惯与生活记录
└─ WF-12 会话结束复盘
        ↓
讯飞数据库 + 讯飞知识库
```

若平台支持工作流嵌套，主流程直接调用子工作流；若不支持，则由主 Agent 调度独立工作流，并以数据库作为共享状态层。

### 4.3 通用工作流协议

通用输入：`user_id`、用户输入、会话标识、当前意图、必要上下文。

通用输出：执行状态、面向用户的回复、结构化结果、建议写入项、下一步动作、错误说明。

标准流程：

```text
开始 → 查询用户状态 → 校验必需信息 → 必要时追问
→ 检索知识库 → 生成结构化结果 → 规则与安全校验
→ 展示结果 → 关键操作由用户确认 → 写入数据库
→ 返回摘要与下一步 → 结束
```

## 5. 功能需求

### 5.1 FR-01 用户建档与初始属性

用户自主设置属性，不采用随机分配作为默认模式。

基础条件包括年级、学校、专业、成绩或排名区间、预算区间、家庭可提供的支持类型、地域流动意愿，以及已有科研、项目、竞赛、实习和学生工作经历。

能力与偏好通过自评和场景题采集，包括学习研究、执行、表达、创造、协作、抗压、风险偏好，以及对稳定、收入、兴趣、城市和学历的重视程度。

系统生成可编辑属性卡，示例：

```text
学业基础：较强；行动执行：中等；表达沟通：较强；
探索意愿：高；风险承受：中等；经济自由度：有限；
时间自由度：中等；现有履历：偏薄。
```

用户确认后才能进入正式模拟。能力字段需标记为“自评”“Agent 推断”或“行为已验证”。没有证据的能力不得包装为事实。

可保留独立“随机重开模式”，仅用于娱乐和理解不同资源条件，不写入真实画像。

### 5.2 FR-02 虚拟大学试玩

真实人生模式使用用户确认的属性，在约 10～15 分钟内完成一轮压缩的大学生涯模拟。系统根据用户当前年级模拟剩余大学阶段，大一用户可体验完整四年。

每学年设置 2～3 个关键事件，覆盖课程、社团、科研、比赛、实习、考试、转向和资源冲突。用户选择会影响时间、精力、资金、能力、履历和机会，但不显示可被机械刷取的精确分值。

结果输出：成长画像、关键选择、获得与放弃的机会、可能的毕业状态、主要风险和重来建议。

### 5.3 FR-03 大学生存大冒险测试

采用场景化题目，如“期末考试前一周如何安排”“社团换届如何选择”“导师项目、实习和比赛冲突时如何决策”。

每个答案同时影响：

- 路径维度：保研、考研、就业、考公、留学。
- 竞争力维度：执行、研究、创造、表达、协作、稳定、适应、风险承受。

结果必须包含主推荐路径、备选路径、核心优势、潜在短板、推荐依据和需要进一步验证的假设，不得只输出单一标签。

### 5.4 FR-04 五大路径推荐

推荐使用可解释的等级制，不展示伪精确录取或成功概率。

```text
推荐得分 = 偏好匹配 + 能力匹配 + 已有履历匹配
           + 资源条件匹配 + 实际行为证据
           - 时间缺口 - 经济压力 - 路径风险
```

输出等级为高匹配、中匹配、待验证、当前不建议投入。每条路径同时输出典型履历要求、当前差距、优先补齐项和备选方案。

### 5.5 FR-05 平行人生模拟

用户创建 2～3 个版本，例如“保研版张三”“就业版张三”“创业/探索版张三”。所有版本使用相同的当前条件和剩余时间，确保可比。

统一比较：路径匹配度、剩余学期轨迹、简历素材质量、技能树广度与深度、时间投入、经济成本、失败风险、可逆性、对其他路径的挤出效应及毕业时选择权。

用户可保存一个版本为主规划。保存、切换或覆盖主规划前必须明确确认，历史版本和切换原因需要保留。

### 5.6 FR-06 真实规划与任务管理

规划层级为“发展路径 → 学期目标 → 月度里程碑 → 本周行动”。每份计划包含成功标准、所需资源、风险、替代方案和“不做清单”。

- 大一“探索者”，绿色：适应、探索、建立习惯，避免盲目堆活动。
- 大二“深耕者”，蓝色：收敛 1～2 条路径，补齐核心课程、技能和项目。
- 大三“破局者”，橙色：聚焦目标，推进科研、实习、考试或申请，建立备选方案。
- 大四“启程者”，金色：完成申请、求职、考试、毕业和成果整理。

### 5.7 FR-07 动态修正

成绩或排名显著变化、新增重要经历、连续任务未完成、预算或地域条件变化、用户主动修改目标、七天试错完成时触发重新评估。

系统必须说明：变化的事实、对原路径的影响、继续/微调/切换建议、机会成本及需要用户确认的动作。不得擅自覆盖主规划。

### 5.8 FR-08 成长反馈与游戏化

完成任务后可更新路径进度、技能经验、连续行动记录、成就徽章和履历素材候选。技能点亮必须绑定行为或成果证据。

技能树包括专业知识、研究、工具技术、表达沟通、协作领导、职业探索、自我管理。

成就包括首次项目、首次公开表达、首次比赛结果、首次实习、连续四周完成核心任务、完成七天试错，以及主动停止低价值投入。系统不设置排行榜、惩罚性扣分或诱导刷任务的虚拟货币。

### 5.9 FR-09 履历素材库与简历条目

用户完成事项后，系统收集背景、目标、职责、行动、工具、结果、量化指标和证明材料位置，生成“行动—方法—结果”式条目。

条目状态包括可直接使用、缺少量化结果、缺少证明材料、需要打磨。素材按目标路径、经历类型、技能和时间筛选。

MVP 输出结构化简历文本；精美文件直接导出属于增强版本。

### 5.10 FR-10 决策分析与七天试错

决策分析输出收益、风险、时间成本、经济成本、机会成本、可逆性、最坏情况和退出条件。

七天试错流程为：提出假设、设置投入上限、制定每日最小行动、记录精力/兴趣/完成度/困难、第七天复盘、决定继续/调整/停止。目的在于获得行为证据，而非七天内完成大型成果。

### 5.11 FR-11 微习惯与生活辅助

通过对话完成散步、冥想、阅读、记账、入门健身和素材整理等微习惯任务。支持连续记录、休息日和补签说明，不将中断描述为失败。

记账保存金额、类别和说明并生成周度汇总；健身根据目标和基础水平提供入门记录与渐进建议。涉及疼痛、疾病、极端节食或高风险动作时停止常规建议并提示专业求助。

MVP 不承诺平台能力之外的系统级主动推送或可视化日历。

### 5.12 FR-12 会话结束复盘

用户表达结束或当前任务完成时，生成：

- 用户复盘：讨论内容、已做决定、接下来三件事、未决问题和下次入口。
- Agent 复盘：新增事实、偏好变化、路径变化、任务状态、待验证推断和推荐开场。

数据库只更新新增或变化字段，不保存无限增长的画像长文，也不将普通闲聊直接视为用户事实。

## 6. 数据设计

### 6.1 用户数据库

| 数据实体 | 关键字段 |
|---|---|
| 用户基础档案 | user_id、昵称、年级、学校、专业、预计毕业时间 |
| 用户画像 | 家庭支持区间、预算、地域偏好、风险偏好、兴趣、优势、能力标签、证据类型 |
| 路径评估 | 五路径等级、竞争力维度、推荐理由、评估时间、触发原因 |
| 平行人生版本 | 版本名、目标路径、学期轨迹、成本、风险、结局摘要、主规划标记 |
| 学期规划 | 学期、核心目标、里程碑、任务、截止时间、优先级、状态 |
| 行动记录 | 任务、投入时间、完成状态、结果、反思 |
| 履历素材 | 类型、组织/项目、职责、行动、工具、结果、指标、证据位置、质量状态 |
| 技能与成就 | 技能、等级、证据、徽章、解锁时间 |
| 会话复盘 | 新事实、状态变化、未决问题、下一次建议 |
| 生活记录 | 习惯、连续天数、支出、健身和微习惯记录 |

数据要求：用户隔离；推断与事实分离；关键变更留痕；支持查看、更正和删除；敏感条件只存规划所需的粗粒度区间。

### 6.2 精选知识库

MVP 收录：五条路径的能力与履历要求、分年级规划、精选学校培养方案与推免规则、通用竞赛/科研/项目/实习/证书、简历范例、学长学姐经验、避坑内容及基础生活指南。

每条知识记录来源、适用学校/专业/年级、更新时间、可信度和内容类型。官方规则、公开资料、经验内容与 Agent 推断必须明确区分。

### 6.3 数据流

```text
用户输入 → 主 Agent 识别身份与意图
→ 查询画像、主规划与近期任务
→ 调用业务工作流 → 检索精选知识库
→ 生成并校验结果 → 用户确认关键选择
→ 写入结构化数据库 → 会话结束复盘
→ 更新画像和下次会话入口
```

## 7. 对话与交互规范

主 Agent 定位为“大学规划教练 + 人生模拟器主持人”。语气年轻、积极、有趣，但不替用户做决定，不过度使用网络梗，也不以制造焦虑促进互动。

复杂任务分阶段提问，每轮优先给出 1～3 个可执行选项。家庭条件、预算和成绩等问题允许跳过。长结果先给结论和对比，再提供展开选项，以适配手机端对话阅读。

关键动作包括确认初始画像、保存或切换主规划、写入重要履历、覆盖计划和删除记录。纯模拟和临时建议不自动写入正式规划。

- 精选路径、学校、专业、项目和竞赛知识库。

## 9. 成功指标

- 建档完成率。
- 虚拟大学试玩完成率。
- 生存大冒险测试完成率。
- 平行版本创建率。
- 主规划保存率。
- 首个学期任务生成率。
- 7 日内再次使用率。
- 完成任务后的履历条目生成率。
- 用户主动修正画像或规划的比例。

不以对话轮数作为核心指标，避免为延长使用时长制造无效交互。

## 10. MVP 验收标准

新用户能够：

1. 建立并修改基础画像。
2. 自主设置和确认初始属性。
3. 完成一轮虚拟大学模拟。
4. 完成场景化路径测试。
5. 获得主路径、备选路径和可解释依据。
6. 创建 2～3 个条件一致的平行版本。
7. 对比履历、技能、成本、风险和机会成本。
8. 保存一个版本为主规划。
9. 获得当前学期计划和首批任务。
10. 下次对话从已保存状态继续。

系统验收要求：用户数据隔离；画像和规划可修改或删除；路径变更留痕；关键写入有确认；写入失败不得声称成功；会话结束可生成结构化复盘。

## 11. 风险与应对

| 风险 | 应对措施 |
|---|---|
| 用户把模拟当作预测 | 使用情景推演语言，持续展示免责声明，不输出伪精确概率 |
| 知识与政策过期 | 记录来源和更新时间，政策类信息强制提示官方复核 |
| 用户画像失真 | 区分自评、推断、已验证，并用实际行为持续校准 |
| 大模型输出不稳定 | 结构化输出、字段校验、缺失检测和失败重试 |
| 工作流节点膨胀 | 模块化工作流、统一输入输出、数据库共享状态 |
| 敏感信息收集过度 | 使用区间、允许跳过、支持查看与删除 |
| 游戏化引发焦虑 | 无排行、无羞辱性失败、允许休息和重新规划 |
| 推荐形成标签 | 输出备选路线和待验证假设，用户可覆盖推荐 |
| 工作流无法嵌套 | 主 Agent 调度独立工作流，数据库承担共享状态 |

## 12. 非功能与安全需求

- 长流程支持分段保存和中断续接。
- 所有工作流采用统一字段命名和版本标识。
- 关键输出包含依据、局限和下一步。
- 娱乐模拟、临时建议和正式规划的数据必须分离。
- 涉及医疗、心理、法律和财务内容只提供一般信息和专业求助提示。
- 产品不承诺保研、录取、就业、薪资或考试结果。
- 用户画像与行为记录遵循最小必要原则。

## 13. 上线前待核实的平台能力

以下事项不改变产品设计，但需要在讯飞星辰平台配置前核实：

1. 工作流是否支持直接调用另一个已发布工作流。
2. 数据库节点支持的字段类型、查询条件、更新方式和单用户隔离能力。
3. 主 Agent 调度多个工作流时的参数传递方式。
4. 单次工作流节点数、执行时长、模型调用次数和上下文长度限制。
5. 是否支持定时或主动消息；若不支持，微习惯提醒改为下次对话待办。
6. 知识库引用来源与更新时间能否在回答中稳定返回。

## 14. 产品边界

产品提供信息整理、情景模拟、规划和行动支持，不替代学校官方政策、职业咨询、心理咨询、医疗建议或用户本人决策。任何路径建议都应被视为基于当前数据的可修正方案。
