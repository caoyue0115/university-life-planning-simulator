# WF-01 Final Consistency Update Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rewrite the WF-01 build and debugging guide so every schema field, node input/output, branch, prompt, SQL statement, message, and test case matches the configuration that passed all three editor tests.

**Architecture:** Keep WF-01 as a single-parameter workflow whose `AGENT_USER_INPUT` contains a flat JSON string. Use `user_key` as the business identity while preserving the platform-owned `uid`; keep confirmation tokens internal to the database for optimistic write protection, and confirm the latest pending draft from database state instead of requiring users to copy tokens.

**Tech Stack:** Markdown, Mermaid, iFlytek Xingchen workflow nodes, Python code-node snippets, SQL.

## Global Constraints

- Modify `docs/workflows/WF-01-user-profile.md`, its two generated PNG diagrams, and this implementation plan.
- Preserve the platform-owned database fields `id`, `uid`, and `create_time`.
- Use `user_key:String` for all business-user reads and writes.
- Keep `AGENT_USER_INPUT:String` as the only start-node parameter.
- Treat `confirmation_token` as an internal database concurrency value; never require it in the three user test inputs.
- Every branch must terminate at N30 through a message node; no dangling lines.
- All code-node return keys must exactly match their declared output variables.
