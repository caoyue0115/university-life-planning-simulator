# WF-02～WF-12 调试与前置依赖教程扩写设计

日期：2026-07-19

## 目标

以 WF-01 的逐节点、逐轮输入、明确分支和数据库核验方式为基准，把 WF-02～WF-12 的调试部分改成新手可以照着操作的完整教程，并把每个工作流依赖的上游工作流和数据记录讲清楚。

## 已确认的做法

采用“每个 WF 独立、具体、可执行”的方案，不采用只有一句话的测试清单，也不把关键步骤挪到一份公共文档后让读者来回跳转。

每份 WF 文档新增或扩写以下内容：

1. 调试前必须准备的上游工作流、数据表和记录。
2. 如何从上游工作流或数据库取得 `profile_json`、`assessment_id`、`comparison_id`、`plan_id`、`task_id`、`trial_id`、token 等变量。
3. 开始节点每个变量的示例值，明确哪些值首轮为空、哪些值第二轮复制。
4. 正常路径的完整节点顺序和数据库预期。
5. 空记录与 SQL 执行失败的区别。
6. 无效输入、安全出口、写入失败、回读失败和取消路径。
7. 临时制造故障的方法、预期消息和测试后的恢复动作。
8. 最终数据库检查与截图留证清单。

## 上游依赖关系

- WF-02：使用 WF-01 已确认画像；状态写 DB-02。
- WF-03：使用 WF-01 已确认画像；过程写 DB-02，完成后写 DB-03。
- WF-04：读取 WF-01 的 DB-01 confirmed 画像、WF-03 的 DB-03 评估，并检索 KB-01。
- WF-05：使用 WF-01 画像和 WF-04 完整推荐；写 DB-04。
- WF-06：读取 WF-05 的 DB-04 比较版本；两轮确认后写 DB-05 active 主规划。
- WF-07：使用 WF-06 的 active `plan_id`；写 DB-06。
- WF-08：读取 WF-06 active 主规划和 WF-07 任务；写 DB-07，但不直接改上游表。
- WF-09：无强制上游工作流，只接受用户真实经历；写 DB-08。
- WF-10：无强制上游工作流；写 DB-09，若决定改主规划则后续进入 WF-06。
- WF-11：无强制上游工作流；写 DB-10。
- WF-12：由主 Agent 传入本次会话文本和已成功写入清单；写 DB-11。

## 验证标准

- WF-02～WF-12 每份文档均含“调试前置准备”“详细调试步骤”“故障恢复”“数据库核验”。
- 所有开始变量、节点编号、表名和输出字段与该 WF 正文一致。
- 所有失败用例明确不得声称已保存。
- 所有临时破坏配置的测试均写明恢复值。
- 文档自动校验脚本通过，流程图不新增悬空分支。
