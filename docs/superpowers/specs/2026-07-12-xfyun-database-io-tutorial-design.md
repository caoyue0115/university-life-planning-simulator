# 讯飞星辰数据库与工作流输入输出教程设计

## 1. 目标

补齐现有 `WF-01` 至 `WF-12` 指南中的数据库、开始输入、节点输入输出和实际配置步骤，使第一次使用讯飞星辰的用户可以从平台首页开始，按教程完成建表、导入字段、配置查询/写入节点、绑定变量、运行调试和检查结果，不需要预先掌握 SQL。

## 2. 已确认的平台事实

### 2.1 数据库默认字段

讯飞星辰创建每张数据表时会自动生成且无法删除：

| 系统字段 | 类型 | 本项目用途 |
|---|---|---|
| `id` | Integer | 每条记录的系统主键 |
| `uid` | String | 平台生成的用户唯一标识，用于用户数据隔离 |
| `create_time` | Time | 记录创建时间 |

导入模板不得重复包含这三个字段。所有查询和写入优先使用平台自动提供的 `uid`，不再额外创建重复的 `user_id` 字段。业务需要更新时间时单独增加 `updated_at`。

### 2.2 导入模板格式

用户提供的 `DB_TABLE_导入模板.xlsx` 已检查，只有一个 `Sheet1`，表头固定为：

```text
字段名* | 数据类型* | 描述* | 默认值 | 是否必填*
```

每张数据表生成一份独立 `.xlsx`，严格保留该表头、工作表名称、表格结构和样式，只在表头下面增加业务字段。字段类型采用平台下拉框可接受的名称；JSON 数据优先保存为 `String`，日期使用 `Time`，计数使用 `Integer`。无法确认的其他类型不写入模板。

### 2.3 数据库节点

截图确认数据库节点支持：

- `自定义SQL`；
- `表单处理数据`；
- 输入参数，支持固定值或引用上游变量；
- SQL 中通过 `{{变量名}}` 引用参数；
- 输出 `isSuccess`、`message`、`outputList`。

教程默认先教 `表单处理数据`，需要一次读取多个字段、排序、限制数量或回读验证时再使用可复制 SQL。字符串变量在 SQL 中写成 `'{{变量名}}'`，非字符串变量不加单引号。

## 3. 数据库架构

采用 11 张共享业务表，而不是每个工作流各建一张重复表。

| 编号 | 表名 | 核心职责 | 主要使用工作流 |
|---|---|---|---|
| DB-01 | `user_profiles` | 正式画像、待确认画像与确认令牌 | WF-01、WF-04、WF-05、WF-08 |
| DB-02 | `simulation_states` | 虚拟大学与场景测试的跨轮续接状态 | WF-02、WF-03 |
| DB-03 | `route_assessments` | 生存测试结果与五路径推荐 | WF-03、WF-04 |
| DB-04 | `parallel_versions` | 平行人生版本及比较结果 | WF-05、WF-06 |
| DB-05 | `main_plans` | 主规划、历史版本、待确认切换 | WF-06、WF-08 |
| DB-06 | `semester_tasks` | 学期、月度、周任务与行动记录 | WF-07、WF-08、WF-09 |
| DB-07 | `growth_reviews` | 成长复盘、动态修正建议与确认状态 | WF-08 |
| DB-08 | `resume_entries` | 履历事实、待确认条目与正式履历 | WF-09 |
| DB-09 | `decision_trials` | 决策分析、七天试错计划和每日记录 | WF-10 |
| DB-10 | `habit_logs` | 微习惯、记账、健身和安全标记 | WF-11、WF-08 |
| DB-11 | `session_recaps` | 会话变化、未决事项和下次入口 | WF-12、总编排 |

每张表都保留系统的 `id`、`uid`、`create_time`。需要区分同一用户的多个业务对象时，增加字符串业务键，例如 `plan_id`、`task_id`、`trial_id`；这些键不是系统主键。

## 4. 仓库文件结构

```text
docs/database/
├─ README.md
├─ DATABASE-SCHEMA.md
├─ SQL-EXAMPLES.md
├─ WF-DATABASE-MATRIX.md
└─ import-templates/
   ├─ DB-01-user-profiles.xlsx
   ├─ DB-02-simulation-states.xlsx
   ├─ DB-03-route-assessments.xlsx
   ├─ DB-04-parallel-versions.xlsx
   ├─ DB-05-main-plans.xlsx
   ├─ DB-06-semester-tasks.xlsx
   ├─ DB-07-growth-reviews.xlsx
   ├─ DB-08-resume-entries.xlsx
   ├─ DB-09-decision-trials.xlsx
   ├─ DB-10-habit-logs.xlsx
   └─ DB-11-session-recaps.xlsx
```

- `README.md`：从讯飞星辰首页进入数据库、建表、填写名称和简介、导入字段、确认默认字段、保存和测试的逐屏教程。
- `DATABASE-SCHEMA.md`：11 张表的每个字段、类型、必填、默认值、使用者和示例。
- `SQL-EXAMPLES.md`：按平台变量语法提供可复制的查询、插入、更新、回读验证和空结果处理。
- `WF-DATABASE-MATRIX.md`：把 WF-01～WF-12 的每个数据库节点映射到表、操作、输入、输出和 SQL。
- `import-templates/`：严格基于用户模板生成的 11 份可直接上传文件。

## 5. 每个 WF 文件的新增教程结构

每个 `docs/workflows/WF-XX-*.md` 增加并统一以下章节：

### 5.1 开始前准备

- 本工作流依赖的数据库表及导入文件链接。
- 必须先完成的上游工作流。
- 开始节点当前能看到的系统输入。
- `uid`、`AGENT_USER_INPUT`、业务键从哪里取得。

### 5.2 开始节点输入

使用表格逐项说明：

| 参数名 | 类型 | 是否必填 | 在哪里添加/选择 | 调试值 |
|---|---|---|---|---|

不得只列变量名，必须说明用户要点击开始节点还是由主 Agent 调用时传入。平台不能直接暴露 `uid` 时，教程提供固定测试值 `test_user_001`，并明确只用于调试。

### 5.3 数据库节点逐项配置

每个数据库节点独立写：

1. 在画布左侧哪个分类找到“数据库”。
2. 拖到哪个节点后面并如何连线。
3. 节点重命名为什么。
4. 选择“自定义SQL”还是“表单处理数据”。
5. 选择哪张数据表。
6. 输入区要删除、保留或新增哪些参数。
7. 每个参数选择固定值还是引用，以及引用哪个上游节点输出。
8. 完整 SQL 或表单条件。
9. `isSuccess`、`message`、`outputList` 分别表示什么。
10. 下一个节点如何引用这些输出。
11. `isSuccess=false`、执行成功但 `outputList=[]`、正常有记录三种情况分别走哪里。

### 5.4 节点输入输出映射

每个节点均提供：

| 当前节点 | 输入参数 | 引用来源 | 输出参数 | 下游使用者 |
|---|---|---|---|---|

大模型提示词中的变量引用必须与表格完全一致。

### 5.5 结束节点输出

列出完整 `result_json` 字段、类型、示例，以及在结束节点界面中选择哪个上游节点的哪个输出。

### 5.6 调试教程

必须说明：

1. 点击右上角“调试”。
2. 输入什么测试文本。
3. 使用什么固定测试 `uid`。
4. 在节点执行详情中查看哪个输入和输出。
5. 如何判断空查询是新用户而不是错误。
6. 如何确认数据库实际增加或更新记录。
7. 调试结束后如何清理测试数据。

## 6. SQL 与数据安全规则

- 所有读写条件必须包含 `uid='{{uid}}'`，防止读取其他用户数据。
- 更新和删除还必须包含业务键或系统 `id`，不得只按 `uid` 批量覆盖。
- 字符串必须加单引号，Integer 不加单引号。
- JSON 字符串在写入前经过变量提取和代码校验；SQL 中的转义方式以平台实际执行为准。
- 教程优先使用参数引用，不允许把真实用户输入直接拼进 SQL。
- `isSuccess=true` 且 `outputList=[]` 表示查询成功但没有记录；不得当成读取失败。
- 关键写入后检查 `isSuccess`，条件允许时再按 `uid + 业务键` 回读验证。

## 7. 生成和验证导入文件

使用 `@oai/artifact-tool` 导入用户模板，复制其 `Sheet1` 和表格结构，为 11 张表分别填入业务字段，再导出 `.xlsx`。生成后执行：

1. 检查每个文件只有一个 `Sheet1`。
2. 检查表头严格等于用户模板。
3. 检查不包含 `id`、`uid`、`create_time`。
4. 检查每个字段名唯一且为英文 snake_case。
5. 检查字段类型只使用平台支持的类型名称。
6. 检查必填列使用平台模板接受的值。
7. 渲染 11 份表格并逐一确认无截断、错列或空白数据行。

## 8. 交付验收

- 新用户可以从平台数据库入口开始，按教程创建 11 张表。
- 11 份 Excel 可以直接用于“导入字段”，且不会重复系统默认字段。
- WF-01～WF-12 均有完整的开始输入、节点输入输出、数据库节点和结束输出教程。
- 每个 SQL 都标明放在哪个数据库节点、输入参数在哪里添加。
- 所有数据库查询说明空结果和执行失败的区别。
- 所有正式写入说明确认、`isSuccess` 检查和回读验证。
- 总编排文档说明主 Agent 如何传递 `uid`、`AGENT_USER_INPUT` 和业务上下文。
