# 讯飞星辰工作流搭建指南文档设计

## 1. 目标

为《大学人生规划模拟器》PRD 中的 `WF-01` 至 `WF-12` 分别编写可直接照做的讯飞星辰工作流搭建指南。读者无需自行把产品逻辑翻译成平台节点，只需按文档拖拽节点、连接、填写变量和提示词、调试并验收。

## 2. 平台边界

指南只使用用户提供的讯飞星辰工作流编辑器截图中已经确认存在的节点：

- 基础节点：大模型、代码、问答节点。
- 工具：工作流、插件、MCP、RPA。
- 知识与数据：知识库、知识库 Pro、长期记忆写入、长期记忆检索、数据库。
- 逻辑：决策、分支器、迭代、Agent 智能决策。
- 转换：变量存储器、变量提取器、文本处理节点。
- 其他：消息。
- 系统节点：开始、结束。

文档不得杜撰平台中不存在的节点名称。产品逻辑中的“校验”“状态判断”“格式整理”等动作，必须明确映射到上述实际节点，例如使用代码、决策、分支器或变量提取器实现。

平台具体字段、按钮或数据库操作能力如无法从截图或实测确认，必须标记为“以当前编辑器显示为准”，并给出不依赖该能力的降级搭法，不得编造界面选项。

## 3. 文档组织

```text
docs/workflows/
├─ README.md
├─ WORKFLOW-TEMPLATE.md
├─ SHARED-CONTRACTS.md
├─ WF-01-user-profile.md
├─ WF-02-virtual-university.md
├─ WF-03-survival-adventure.md
├─ WF-04-path-recommendation.md
├─ WF-05-parallel-lives.md
├─ WF-06-main-plan.md
├─ WF-07-semester-tasks.md
├─ WF-08-growth-review.md
├─ WF-09-resume-entry.md
├─ WF-10-decision-and-trial.md
├─ WF-11-micro-habits.md
└─ WF-12-session-recap.md
```

- `README.md`：推荐搭建顺序、模块关系、端到端业务逻辑和联调说明。
- `WORKFLOW-TEMPLATE.md`：以后新增工作流时可复制的标准写作模板。
- `SHARED-CONTRACTS.md`：统一变量、JSON 输出、确认规则、数据库规则和失败处理。
- `WF-01` 至 `WF-12`：每个小工作流一份独立搭建指南。

## 4. 单个工作流指南结构

每份指南固定包含以下内容：

1. 本工作流解决什么问题，以及何时由主 Agent 调用。
2. 搭建前准备，包括依赖的数据库表、知识库和上游变量。
3. 完成后的画布结构图，节点名称与画布上的命名完全一致。
4. 节点清单，说明需要从左侧拖入几个什么节点。
5. 从空白画布开始的逐步搭建操作：拖拽位置、连线顺序、节点重命名。
6. 每个节点的配置：输入变量来源、提示词、输出变量、分支条件和数据映射。
7. 可直接复制的完整提示词或代码。
8. 调试步骤：测试输入、逐节点观察项、预期输出。
9. 常见错误及平台内修复方法。
10. 验收清单。
11. 与前后工作流、主 Agent 和共享数据库的衔接关系。

指南中的每一步都采用操作式语言，例如：“从左侧‘基础节点’拖入一个‘大模型’节点，放在开始节点右侧；将开始节点右侧连接点拖到大模型节点左侧连接点。”

## 5. 工作流分层

每个复杂工作流同时提供两层搭法：

- 最小可运行版：优先验证主要输入和输出，节点数量少，便于首次搭建。
- 完整业务版：加入数据读取、确认、分支、写入校验、异常处理和上下游衔接。

以 `WF-01` 为例：

```text
最小可运行版：
开始 → 大模型（生成待确认画像）→ 结束

完整业务版：
开始
→ 长期记忆检索或数据库（读取已有画像）
→ 大模型（提取并合并画像）
→ 变量提取器（提取结构化字段）
→ 决策/分支器（判断缺失、确认或修改）
├─ 缺失或修改：消息/大模型（追问或展示修订画像）→ 结束
└─ 已确认：数据库或长期记忆写入 → 决策（检查写入结果）→ 结束
```

最小可运行版只证明“能生成并展示画像”，不得声称已经完成用户确认或保存。完整业务版只有收到明确确认且写入成功后，才能返回保存成功。

## 6. 共享规则

- 开始节点默认接收平台提供的 `AGENT_USER_INPUT`；需要额外输入时在指南中明确列出。
- 工作流之间统一使用英文 `snake_case` 变量名。
- 大模型结构化输出先经过变量提取或代码校验，再交给分支和数据库节点。
- 用户画像、主规划保存或覆盖、重要履历写入等关键操作必须先确认。
- 推断、自评与行为证据分开存储；缺失信息不得虚构。
- 数据库或长期记忆写入失败时，不得返回“保存成功”。
- 模拟结果和临时建议不得覆盖正式画像或主规划。
- 政策及时效性内容应提示用户通过官方渠道复核。

## 7. 12 个工作流的职责边界

| 工作流 | 独立职责 | 核心产物 |
|---|---|---|
| WF-01 | 建档、修改和确认画像 | `profile_json` |
| WF-02 | 运行压缩版虚拟大学试玩 | `simulation_summary_json` |
| WF-03 | 采集场景题答案并形成竞争力信号 | `adventure_result_json` |
| WF-04 | 生成五路径分级推荐 | `route_recommendation_json` |
| WF-05 | 生成并比较 2～3 个平行版本 | `parallel_versions_json` |
| WF-06 | 保存主规划并形成目标层级 | `main_plan_json` |
| WF-07 | 管理学期、月度和每周任务 | `semester_tasks_json` |
| WF-08 | 根据行动证据进行成长复盘和动态修正 | `growth_review_json` |
| WF-09 | 把经历转成履历条目 | `resume_entry_json` |
| WF-10 | 完成决策分析或七天试错 | `decision_trial_json` |
| WF-11 | 记录微习惯与基础生活事项 | `habit_log_json` |
| WF-12 | 汇总会话变化并生成下次入口 | `session_recap_json` |

每个工作流只负责一个核心产物，通过统一变量和数据库共享状态，避免在单一画布中堆积全部业务逻辑。

## 8. 整体搭建顺序与逻辑

推荐按依赖顺序搭建：

```text
共享变量与数据结构
→ WF-01 用户画像
→ WF-02 虚拟大学试玩
→ WF-03 生存大冒险
→ WF-04 路径推荐
→ WF-05 平行人生
→ WF-06 主规划
→ WF-07 学期任务
→ WF-09 履历条目
→ WF-08 成长复盘
→ WF-10 决策与七天试错
→ WF-11 微习惯
→ WF-12 会话复盘
→ 主 Agent 路由与端到端联调
```

其中 `WF-01` 至 `WF-06` 构成首次使用闭环；`WF-07` 至 `WF-11` 构成长期执行和修正闭环；`WF-12` 负责把每次会话产生的变化压缩为下一次可继续使用的状态。

## 9. 验证标准

- 新手能仅依据文档在讯飞星辰画布中复现节点和连线。
- 所有节点名称均能在用户截图所示的平台节点列表中找到。
- 每份文档至少包含一个成功用例和一个失败或缺失信息用例。
- 关键写入都有确认和写入结果检查。
- 12 个工作流的输入输出能通过共享协议衔接。
- 总览文件能解释首次体验、长期执行、动态修正和会话续接四条逻辑链。

## 10. 已知风险与处理

- 平台版本差异：界面字段可能变化，因此以功能和变量映射为主，并注明需以当前编辑器显示为准。
- 工作流嵌套能力未确认：优先提供“主 Agent 调用独立工作流”的方案；若平台支持“工作流”节点，再补充画布内调用方式。
- 数据库字段能力未确认：先定义逻辑数据结构，具体建表操作在实测后补充；指南同时提供长期记忆方案作为降级路径。
- 长流程中断：复杂流程拆成可独立调用的小工作流，状态通过数据库或长期记忆保存。
